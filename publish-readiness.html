TangentBar — publish readiness

What stands between today's dev binary and something strangers can download · 2026-07-05

Verdict

Not publishable yet — and the gaps are structural, not polish. The core UX is proven (define, chat, local-first speed), and the privacy story is genuinely strong (everything on-device, network = localhost only). But today TangentBar is a bare terminal binary that assumes a running LM Studio with models already pulled, inherits its Accessibility grant from your shell, and logs every clicked word to the system log. Five stages below, in dependency order.

Ship-blocker found during this review The NSLog diagnostics we added while debugging log every extracted word, selection size, and app name — readable by any process via the unified log. Fine for dogfooding; a privacy leak in a shipped build. Must be gated behind a debug flag before any release.
Machine check, just performed Your keychain has one signing identity: "Whorl Dev" — an Apple Development cert, not a Developer ID Application cert. Public distribution outside the App Store requires the latter, which requires Apple Developer Program enrollment ($99/yr, may already exist on the account that issued Whorl Dev). Without it, downloads hit the Gatekeeper wall.

Stages 1–5, in dependency order

1 · Bundle + sign + notarize (the foundation — everything else keys off it)

  • TangentBar.app with Info.plist (LSUIElement, bundle id, icon), built by a small script or SPM plugin.
  • Developer ID cert → codesign --options runtime → notarytool submit → staple. Hardened runtime is mandatory for notarization.
  • This is also what makes the Accessibility grant durable and grantable by normal users.

2 · Onboarding (your "setup widget") — a 3-step first-run window

  1. Welcome + Accessibility: explain why (reads text around your click; never phones home), deep-link straight to the Accessibility pane, live-detect the grant (the poll already exists — give it UI).
  2. Model setup — the step you called out, detection with three states:
    • server + models found → auto-pick (qwen→gemma preference already built), show it, done.
    • server, no models → pull one in-app with a progress bar. Ollama's REST API (POST /api/pull) streams download progress — no terminal needed. LM Studio equivalent exists (lms get) but is CLI-side.
    • no server → guided install: "Install Ollama" link (one .dmg, no account), relaunch detection, then the pull step above.
  3. Try it: a sample paragraph in the window — "double-click any word" — live demo wired to the real pipeline, plus the drag-to-chat gesture.
Architectural call: Ollama as the blessed bootstrap path Supporting detection of both servers stays, but the guided path should bless exactly one (Ollama: single dmg, REST pull API, no login) rather than presenting a chooser to someone who doesn't know what either is. LM Studio users self-serve; the setup widget optimizes for the person with neither. Bundling llama.cpp/MLX in-app would remove the dependency entirely — right call eventually, wrong size for v0.1.

3 · Hardening (small items, all required)

  • Gate the NSLog traces behind --debug / a config flag (ship-blocker above).
  • Gate the claude fallback on CLI presence (v1's resolve_model lesson) — for most users the "fallback" should be a clear "model unavailable" state, not a 16 s mystery hang.
  • Settings surface for excludedApps (the Slay-the-Spire lesson) — even just "Exclude frontmost app" in the menu.
  • "Start at Login" via SMAppService + the toggle.
  • 3b ⌘C synthesis: restore all pasteboard flavors (TODO already in code), and disclose the mechanism in docs.
  • License + repo visibility decision (below).

4 · Distribution

  • GitHub Releases with the notarized .dmg (or zip) — the canonical "easy download".
  • Homebrew cask (brew install --cask tangentbar) — cheap to add once releases exist.
  • Auto-update via SparkleSparkle — the de-facto open-source auto-update framework for non-App-Store Mac apps: the app checks an appcast feed and self-updates with a signed delta., or defer and let Homebrew carry updates for v0.1.

5 · Docs + site

  • README: install, the two gestures, permissions rationale, privacy statement ("your text never leaves the machine — the only network calls are to localhost"), troubleshooting (no pill? → check Accessibility; slow first answer? → model cold load).
  • Public compatibility matrix distilled from ADAPTERS.md (works great / works / not yet: Google Docs, scanned PDFs → the OCR rung is the eventual answer).
  • Reskin v1's website (already Vercel-configured) with the v2 story + download link.
