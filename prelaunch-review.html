Verdict

The codebase is in genuinely good shape — clean build with zero warnings, a disciplined four-layer architecture (sensor → extract → surface → engine), and hard-won platform knowledge captured as comments where it matters. Nothing structural needs to change before publishing.

What stands between here and a downloadable GitHub release is about 1½–2 focused days: four release blockers (word-logging, app bundle, the no-model first-run state, README/LICENSE), plus three P1 bugs found in this review that are cheap to fix and embarrassing to ship — a password-field privacy gap, a pipe deadlock that can hang the panel forever, and a stream bug that truncates output.

Framing choice Your ask — "downloadable even with simple warnings" — maps to the free path from the Jul 10 distribution doc: no $99 Apple Developer Program, ad-hoc signed. One honest correction to "simple warnings": since macOS 15, a plain-zip download of an ad-hoc app shows "damaged and can't be opened" with no Open Anyway button — that's a wall, not a warning. The curl | sh installer avoids the quarantine attributeQuarantine attribute — an extended file attribute browsers stamp on downloads. Gatekeeper only inspects files carrying it; curl doesn't set it, so script installs never hit the Gatekeeper dialog at all. entirely, so it opens clean. Decision D1 below.

Bugs found in this review

Severity: P1 = fix before publishing · P2 = fix soon, not gating · P3 = known/acceptable, document it.

SevBugWhere
FIXED Web password fields aren't suppressed. The secure-field bail checks role == "AXSecureTextField" only. Browsers expose password inputs as AXTextField with subrole AXSecureTextField — so a double-click on a browser password field can run the full ladder, including the ⌘C-synthesis rung, and (today) log the result. Check AXSubrole too, at every bail site. Extractor.swift:106‑109, 275, 335
FIXED Unread stderr pipes can deadlock the claude fallback. process.standardError = Pipe() with no reader: if claude writes more than the ~64 KB pipe buffer to stderr (auth errors, verbose failures), the process wedges and terminationHandler never fires — the panel hangs on "asking claude…" forever. Same pattern in TerminalContext.run (mitigated there by the 1.5 s kill, but the stderr data is still lost for diagnosis). Drain or null-device stderr. Engine.swift:147 · TerminalContext.swift:39
FIXED filterThink drops held-back text at stream end. To tolerate <think> tags split across chunks, up to 8 trailing chars after a < are held in carry — but nothing flushes carry when the stream completes. Any answer whose final characters include a < (math, code, HTML in a definition) gets silently truncated. Flush carry in didCompleteWithError. SSEStream.swift:61‑65, 86‑89
P2 Clipboard rescue can destroy a non-text clipboard. Rung 3b saves only the string flavor before synthesizing ⌘C. If the user's clipboard held an image or file, restore writes nothing back — their clipboard is silently wiped. The TODO is already in the code; fix is to snapshot all pasteboardItems flavors. Also worth a one-line disclosure in the README (clipboard managers will record the synthetic copy). Extractor.swift:369‑396
P2 Triple-click and double-click-drag both fire a define. The trigger is mouseUp with clickState == 2, which also matches the second click of a triple-click (select-paragraph) and the end of a double-click-drag (select-by-words). Users who triple-click to select a line get a surprise definition panel. Cheap partial fix: ignore when the up-point traveled >8 px from down (drag case); the triple-click case needs a ~250 ms defer to see if a third click lands. EventTap.swift:39
P2 A pure-reasoning response triggers a spurious claude fallback. receivedAny is only set when visible text survives think-stripping — a response that's all <think> (small models sometimes burn the whole token budget thinking) reads as "server gave nothing" and silently burns a 16 s claude call. Track raw-chunks-received separately from visible-chunks. SSEStream.swift:53‑57 · Engine.swift:117‑124
P3 20 s idle timeout can kill slow local models. timeoutIntervalForRequest = 20 is an idle timeout — a big model that takes >20 s to first token gets cancelled → fallback, even though the server was fine. Unlikely with your models (think-tokens keep the stream alive); worth a comment and maybe 60 s for the chat path where users pick bigger models. SSEStream.swift:23
P3 The Chromium nudge retry costs 150 ms on every textless click, in every app. extract(at:) retries once after usleep(150_000) even for apps that were never Chromium. Acceptable latency tax for v0; gate on bundle-id prefix if it ever bothers you. Extractor.swift:312‑316
The standing ship-blocker is still open AppController.swift:109 and :201 log every extracted word (and app name) via NSLog — readable by any process in the unified log. Flagged in both prior reviews; must be gated behind a --debug flag (or migrated to os.Logger with %{private} redaction) before anything ships. The other NSLog sites only log coordinates, ladder names, and character counts — those are fine to keep.

Code quality opportunities (non-gating)

  • Rename provider = "lmstudio" → "local". It actually means "any local OpenAI-compatible server" — Ollama rides the same path. The misnomer will confuse the first outside contributor. One-line config migration: treat "lmstudio" as "local" on load.
  • ModelDiscovery.server names by port-sniffing (:11434 → "Ollama") — a custom base URL on another port shows as "LM Studio". Cosmetic.
  • Chat model is never prewarmed. prewarm pings only tangentModel; picking a separate (bigger) chat model means the first chat turn eats the ~7.6 s cold JIT load. Ping the chat model too when it's set.
  • Excerpt.focus pads around the first occurrence of the selection in the source; repeated phrases can pad the wrong instance. Rare; noting for the record.

Assumption — audience Carrying forward the Jul 10 call: the v0 audience is technical-adjacent and comfortable pasting one terminal command. That's what makes the curl installer viable as the primary channel. If you want non-technical downloaders, no free path serves them well — that audience needs the $99 notarization route, out of scope for this cut.
Architectural choice — no Xcode project Staying pure SwiftPM with a shell-script bundler, no .xcodeproj. Keeps the repo clean and CI trivial; costs nothing until you need entitlements or Sparkle, both of which the script can absorb later.

D1 · Install channel for the free (unsigned) pathRecommended: both

Ad-hoc zip on macOS 15 = "damaged" dialog with no Open Anyway; the escape is one xattr -dr com.apple.quarantine command. The curl installer never acquires quarantine, so it opens clean — and it's the only channel structurally insulated from Apple tightening Gatekeeper further.
