TangentBar — architecture

The four layers (sensor → extraction → surfaces → engine) and the two flows that cross them. Source of truth: architecture.mmd in the repo root.

The two flows

Flow A — define Double-click a word → EventTap → Extractor climbs the word + context ladder → TangentPanel opens at the cursor (no pill by default, D8) → Engine streams a 2–4 sentence definition.
Flow B — selection chat Drag-select a passage → EventTap → selection grab → an “ask about …” pill → ChatPanel (activates the app so Wispr Flow dictation works) → Engine streams each turn.
normal path fallback / conditional (local server unreachable → claude CLI)

Reading notes

  • Local-first (D7): ModelDiscovery probes LM Studio and Ollama at launch and every time the menu opens; qwen beats gemma beats the rest. Claude models (haiku / sonnet / opus) are appended as first-class picks whenever the claude CLI is found at one of six known install paths — pickable any time, never the auto-selected default.
  • Two model slots: defines want tiny-and-instant (~0.16 s warm, 300 tokens); chats get their own, smarter model (700 tokens, 60 s first-token budget). “Chat Model: same as define” is the default.
  • Fallback honesty: when the local server gives nothing, the panel says “unreachable — asking claude…” rather than hanging; when a model burns its whole budget inside <think>, the panel says that too instead of lying about reachability.
  • Prewarm: the engine pings the selected model at launch and every 240 s with ttl: 7200, so LM Studio keeps the JIT model loaded — the difference between 0.16 s and 7.6 s.
