TangentBar — pre-built distribution

The "simple version": strangers download a pre-built app and run it without you paying Apple — what that actually takes, and the decisions it forces. · 2026-07-10

Verdict

Yes, this is possible — but "ignore the warning" no longer means clicking past a dialog. Since macOS 15 Apple removed the right-click → Open bypass. An app that isn't notarizedNotarization — Apple's automated malware scan for apps distributed outside the App Store. You upload the app, Apple scans and blesses it, and Gatekeeper then opens it without complaint. Requires the $99/yr Developer Program. hits one of two walls when a stranger downloads it, and the wall depends on how it was signed:

So the free path is real, but the install instructions are the product: a one-line curl installer sidesteps Gatekeeper legitimately (no quarantine is ever applied), while a plain zip download forces every user through a Terminal incantation or a buried Settings toggle.

The second, bigger wall isn't Apple — it's that the app currently assumes your machine: a running LM Studio with qwen3.5-0.8b-mlx pulled, Ollama as backup, and the claude CLI for fallback. A stranger has none of these; today they'd get a panel that silently hangs and then errors. The Jul 5 publish-readiness review covers the full productization ladder; this doc scopes only what the free download-and-run cut needs.

Assumption — who's downloading this I'm assuming the v0 audience is technical-adjacent (comfortable pasting one Terminal command). That's what makes the curl-installer path viable as the primary channel. If you want non-technical users, the free tier can't serve them well — that audience needs D1 option D (pay Apple).
Gatekeeper behavior is a moving target The table above is accurate as of macOS 15/26, but Apple has tightened unsigned-app handling every year. The curl-script channel is the only one structurally insulated from further tightening (no quarantine → Gatekeeper never evaluates). Worth re-verifying the exact dialog wording on a current OS before writing the README.

What the stranger experiences (recommended path)

  1. Runs the documented one-liner: curl -fsSL https://…/install.sh | sh → script downloads the release tarball, expands to /Applications/TangentBar.app, opens it. No quarantine, no Gatekeeper dialog.
  2. App launches → macOS asks for Accessibility → System Settings opens on the right pane → grant → app detects it (poll flow already built) and arms.
  3. No model server found → menu-bar state points at Ollama install; once Ollama is running, discovery auto-picks a model (qwen→gemma preference already built).
  4. Double-click a word anywhere → definition panel. Done.

Architectural choice — the installer script is the product's front door Under the recommended stack (A/A/A/A/A), the install script carries real responsibility: download the right release, expand to /Applications, remove any stray quarantine, launch, and print the two next steps (Accessibility, Ollama).
